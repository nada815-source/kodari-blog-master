### 1. 🚨 주요 트러블슈팅 및 해결 방안

#### ① 서버 실행 경로 문제 (가장 중요!)
- **현상:** 코드를 수정해도 브라우저에 반영되지 않음.
- **원인:** 터미널이 `C:\Users\JK` 경로에서 서버를 켜려고 시도함 (프로젝트는 D드라이브에 있음).
- **해결:** 반드시 아래 순서대로 서버를 실행해야 합니다.
  1. `d:` 입력 후 엔터
  2. `cd d:\초보프로젝트\blog-generator` 입력 후 엔터
  3. `npm run dev` 입력

#### ② API 키 인식 및 SDK 오류
- **현상:** `An API Key must be set when running in a browser` 에러 지속 발생.
- **원인:** 구글 공식 SDK 라이브러리가 브라우저 환경에서 키를 인식하는 방식이 불안정함.
- **해결:** SDK를 과감히 버리고, **`fetch`를 이용한 API 직접 호출 방식** 도입. 이 방식은 라이브러리 버전 문제에서 100% 자유롭습니다.

#### ③ 모델명 불일치 문제
- **현상:** `models/gemini-1.5-flash is not found` 에러 발생.
- **원인:** 대표님의 API 키는 일반적인 `1.5-flash`가 아닌, 차세대 모델인 **`gemini-2.5-flash`**를 지원하는 특별한 계정이었음!
- **해결:** 자가 진단 로직을 통해 사용 가능 목록을 확인한 후, **`gemini-2.5-flash`** 모델로 최종 매칭 완료.

### 2. 🛠️ 현재 시스템 사양 (Golden Config)
- **엔진:** Gemini 2.5 Flash (차세대 고성능 엔진)
