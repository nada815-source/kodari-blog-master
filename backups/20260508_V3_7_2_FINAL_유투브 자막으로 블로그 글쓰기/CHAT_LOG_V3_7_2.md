## 1. V3.7.0 ~ V3.7.1: '구글 경비병' 우회 작전 (수동 주입 모드)

*   **해결 전략 논의:** 
    1. 로컬 환경(`npm run dev`)에서 직접 돌리기 (불편함 증가)

*   **최종 결정 및 구현 (V3.7.1):** 
    *   모바일/PC 어디서든 안정적으로 구동되는 **[작전 2: 수동 주입 모드]** 채택.
    *   Chrome/Edge 확장 프로그램('YouTube Summary')과 연동하여, 사용자가 클릭 한 번으로 복사한 자막을 붙여넣을 수 있는 대형 Textarea UI 신설.
    *   서버리스 의존성을 완전히 제거하고 프론트엔드 단독으로 안정성 100% 확보.

## 2. V3.7.2: True-Visual 렌더링 버그 완전 소탕
*   **이슈 발생:** 사용자가 붙여넣은 자막으로 블로그 글 생성은 성공했으나, 화면(Screen) 미리보기 영역에 `<strong class="font-bold">` 등의 HTML 태그가 그대로 노출되는 렌더링 버그 발견.
*   **원인 분석 (투트랙 엔진의 오해):**
    *   **클립보드 엔진 (정상):** 복사 버튼을 누를 때 작동하는 `convertMarkdownToHtml`은 백그라운드에서 인라인 스타일로 완벽하게 변환하여 클립보드에 담아줌 (네이버 블로그 붙여넣기 시 완벽 적용됨 확인).
    *   **화면 엔진 (오류):** 화면을 그리는 `ReactMarkdown` 컴포넌트에 HTML 태그를 해석하는 `rehype-raw` 플러그인이 누락되어, 강조 태그가 문자열(외계어)로 그대로 출력됨.
*   **최종 조치 (V3.7.2):**
    *   `npm install rehype-raw` 설치 및 `ReactMarkdown` 컴포넌트에 플러그인 장착 완료.
    *   표(Table) 컴포넌트가 화면에서도 아름답게 보이도록 Tailwind CSS(prose-table) 덧칠.
    *   화면 출력(미리보기)과 클립보드 복사(결과물) 모두 100% 완벽한 시각적 만족도 달성.

## 3. 대표님의 '매의 눈'과 투트랙 엔진의 재발견
